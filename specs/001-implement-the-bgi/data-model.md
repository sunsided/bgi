# Data Model: BGI Graphics API

## Core Entities

### GraphicsContext
- **Purpose**: Central graphics session management with complete BGI state
- **Key Attributes**:
  - current_position: Point (x, y coordinates for drawing operations)
  - current_color: Color (active drawing color from palette or RGB)
  - line_style: LineStyle (solid, dotted, dashed, custom patterns)
  - fill_style: FillStyle (solid, patterns, custom textures)
  - viewport: Viewport (clipping rectangle and coordinate system)
  - font_settings: FontSettings (typeface, size, direction, justification)
  - backend: Box<dyn Backend> (rendering implementation)
- **Relationships**: Owns Backend instance, manages WindowState
- **State Transitions**: Uninitialized → Initialized (via initgraph) → Closed (via closegraph)

### Backend (Trait)
- **Purpose**: Abstraction for different rendering implementations (winit, future SDL, etc.)
- **Key Operations**:
  - Window management (create, resize, close, refresh)
  - Drawing primitives (pixel, line, rectangle, circle, arc, ellipse, polygon)
  - Text rendering (with font support and positioning)
  - Input handling (keyboard events, mouse events)
  - Graphics mode management (resolution, color depth)
- **Relationships**: Implemented by concrete backend types (WinitBackend, etc.)

### Color
- **Purpose**: BGI-compatible color representation supporting both palette and RGB modes
- **Key Attributes**:
  - Indexed(u8): Classic 16-color BGI palette (BLACK, BLUE, GREEN, etc.)
  - Rgb(RgbColor): Modern ARGB color extensions
- **Validation Rules**: Palette indices must be 0-15 for BGI compatibility
- **Relationships**: Used by all drawing operations and graphics state

### GraphicsMode
- **Purpose**: Display resolution and color depth configuration
- **Key Attributes**:
  - mode_id: ModeId (CGA, EGA, VGA, custom)
  - resolution: (width: u32, height: u32)
  - color_depth: ColorDepth (4-bit, 8-bit, 24-bit)
  - driver: GraphicsDriver (auto-detection or specific backend)
- **Validation Rules**: Mode must be supported by active backend
- **State Transitions**: Available → Active (via initgraph) → Inactive (via closegraph)

### DrawingState
- **Purpose**: Current graphics settings affecting all drawing operations
- **Key Attributes**:
  - position: Point (current pen position)
  - color: Color (active drawing color)
  - line_style: LineStyle (line appearance and thickness)
  - fill_pattern: FillPattern (area fill appearance)
  - write_mode: WriteMode (copy, XOR, AND, OR pixel operations)
- **Relationships**: Contained within GraphicsContext
- **Validation Rules**: All settings must be valid for current graphics mode

### Viewport
- **Purpose**: Clipping rectangle and coordinate system management
- **Key Attributes**:
  - bounds: Rect (x1, y1, x2, y2 clipping rectangle)
  - clip_enabled: bool (whether clipping is active)
  - origin: Point (coordinate system offset)
- **Validation Rules**: Bounds must be within graphics mode resolution
- **Relationships**: Referenced by all drawing operations for clipping

### FontSettings
- **Purpose**: Text rendering configuration with BGI font support
- **Key Attributes**:
  - font_type: FontType (DefaultFont, TriplexFont, SmallFont, etc.)
  - size: FontSize (character size multiplier or pixel height)
  - direction: TextDirection (horizontal, vertical)
  - justification: TextJustification (left, center, right)
- **Validation Rules**: Font type must be available in current backend
- **Relationships**: Used by text rendering functions

### InputEvent
- **Purpose**: Unified keyboard and mouse event representation
- **Key Attributes**:
  - KeyPress { key: KeyCode, modifiers: KeyModifiers }
  - KeyRelease { key: KeyCode, modifiers: KeyModifiers }
  - MouseMove { x: i32, y: i32 }
  - MouseClick { x: i32, y: i32, button: MouseButton }
- **Validation Rules**: Coordinates must be within current viewport
- **Relationships**: Generated by Backend, consumed by input handling functions

### WindowState
- **Purpose**: Window management and display properties
- **Key Attributes**:
  - id: WindowId (unique window identifier)
  - title: String (window title bar text)
  - visible: bool (window visibility state)
  - size: (width: u32, height: u32)
  - pixel_buffer: Vec<u32> (ARGB pixel data)
- **State Transitions**: Created → Visible → Hidden → Destroyed
- **Relationships**: Managed by Backend implementation

## Entity Relationships

```
GraphicsContext
├── owns Backend (trait object)
├── contains DrawingState
├── contains Viewport
├── contains FontSettings
└── manages WindowState

Backend (trait)
├── implemented by WinitBackend
├── generates InputEvent instances
└── renders to WindowState pixel buffer

DrawingState
├── uses Color for drawing operations
└── references current GraphicsMode constraints

Viewport
└── applies clipping to all drawing operations

Color
└── supports both BGI palette and RGB modes
```

## BGI API Mapping

Each BGI function maps to operations on these entities:
- `initgraph()` → Creates GraphicsContext with specified GraphicsMode
- `setcolor()` → Updates DrawingState.color
- `moveto()` → Updates DrawingState.position
- `lineto()` → Draws line from current position using DrawingState
- `circle()` → Backend drawing operation with current DrawingState
- `getch()` → Waits for InputEvent from Backend
- `setviewport()` → Updates Viewport bounds and clipping
